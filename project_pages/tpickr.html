Task Pickr (Redesign)

To-do list that picks a task for you start with

Details

I made the first Task Pickr as a project after discussing with others how I wished to-do apps could pick a task for me to start with. It was built with HTML, CSS, Bootstrap, and Vanilla JS, and for the design, I wanted to try making different themes.

Task Pickr #1

I wasn’t fully happy with the final look. Not just in design but also in function. The list and picker were finicky in how they worked due to how the list deleted items. Also, there were readability issues with the font. After revisiting the project, researching, and playing with the site, I knew I wanted to:

  1. Redesign it to something simpler and more neutral. I liked the bubbles’ design and was ok with the night theme but decided on something more gender-neutral. Also, I wasn’t completely happy with the font choice.
  2. Remove high and low-priority sections. After researching more on people with analysis paralysis and neurodivergence, and even asking close family/friends, not everyone can easily distinguish which tasks are more important outside of deadlines. Sometimes, even deadlines aren’t enough to push priority, so removing those sections would give more focus to the picker.
  3. Only pick one task instead of two. No high/low- priority, no need for two tasks. Also, it’s easier for those to focus on one task at a time.
  4. Add edit functionality. The first TP didn’t allow for the editing of tasks. The second one would.

I also wanted to reduild it using React to help with compartmentalizing some of the code.

Design:

For the new design, I kept the structure from the first TP and made the necessary changes. I also wanted to keep it to no more than 3 colors, not including white.

Code:

With the code, I referenced the first Task Pickr but made changes to how the list was built. Since I could use React, I could make separate components for the task list and picker. Each input is added to an array with a unique ID using NPM’s UUID package. The first TP used the item’s array placement as an ID which did lead to issues in deletion. The edit function calls a popup with the task to be edited in the input. This allows users to make sure they're editing the right text before submitting.

Tools Used

React, HTML/CSS, Bootstrap, Linearity Curve, VS Code
